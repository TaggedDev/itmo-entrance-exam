# AI Usage

AI использовался для декомпозиции задания, выбора минимального PoC-скоупа и проверки
архитектурных компромиссов. Финальное решение оставить Chroma, LangChain, DeepSeek и
self-hosted Langfuse принято как достаточно близкое к целевой архитектуре, но без
production highload-сложности.

AI помог сформировать REST-контракты, структуру Docker Compose, список документов,
smoke-сценарии и реализацию линейного ML pipeline.

Часть предложений была сознательно упрощена: LangGraph не используется, отдельная ML-модель
не обучается, очереди и асинхронная генерация не добавлены. Вместо этого PoC показывает
главный путь: redaction -> DeepSeek classification -> Chroma retrieval -> grounded answer.

Тесты не вызывают внешний DeepSeek API: LLM подменяется fake-компонентом, чтобы проверка
оставалась быстрой и воспроизводимой.
